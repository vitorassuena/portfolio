Vehicle Planning Overhaul

Planning the correct amount and type of cargo vehicles is one of the most important tasks in a logistics company, and doing so in a timely manner is even more important. This project aimed to completely optimize this proccess and allow the analysts to plan different scenarios in minutes.
Tools: Python SQL Streamlit

Problem description

The Vehicle Planning team faced a significant challenge in determining the optimal quantity and type of vehicles to allocate to each facility. This process consumed an entire day every week for each analyst, who manually analyzed historical data and client forecasts to create individual routes.

With over 1,500 routes per week, this manual process was time-consuming and prone to errors. The team needed a solution that could automate the vehicle allocation process, account for various constraints, and seamlessly integrate with their internal transport planner system.

Features

  1. Data Ingestion: The project leveraged historical data from the company's database and client forecasts stored in spreadsheets. By automating the data ingestion process, analysts were relieved from manually extracting and organizing data.

  2. Calculation Engine: The script incorporated a robust calculation engine that determined the appropriate vehicle type and quantity for each facility. It considered multiple factors such as historical demand, client forecasts, facility-specific restrictions (e.g., vehicle type per facility), and day of the week requirements.

  3. Route Spreadsheet Generation: The Python project generated a comprehensive spreadsheet where each row represented a different route. This spreadsheet encompassed all the necessary details, including origin, destination, vehicle type, and quantity. The automated generation eliminated the need for analysts to manually create each route, saving significant time and reducing human errors.

  4. API Integration: The project seamlessly integrated with the company's internal transport planner system via an API. The generated route spreadsheet was automatically imported into the system, further streamlining the planning process. This integration eliminated the tedious task of manually inputting each route into the system, resulting in substantial time savings.

Key Results

  1. Time Savings: The automated solution saved over 6 hours weekly for each analyst, who previously dedicated their entire Friday to the vehicle planning process. This time could now be utilized for other value-added tasks, enhancing overall productivity.

  2. Increased Accuracy: By eliminating manual data manipulation and route creation, the project significantly reduced human errors. The automated calculation engine ensured consistent and accurate allocation of vehicles, optimizing the logistics operations.

  3. Scalability: The Python project successfully handled over 1,500 routes per week, accommodating the growing demands of the organization. As the company expanded, the solution remained scalable and adaptable to changing requirements.

  4. Streamlined Scenario Analysis: The Python project enabled effortless creation and analysis of different scenarios, optimizing vehicle allocation during peak demand. This streamlined process facilitated quick decision-making, ensuring efficient resource utilization and timely deliveries for clients.

Conclusion

By leveraging Python's flexibility and powerful libraries, this logistics unicorn successfully transformed its vehicle planning process. The automated solution not only saved valuable time for analysts but also improved accuracy and scalability.

By ingesting data, performing complex calculations, and generating route spreadsheets, the project streamlined operations and integrated seamlessly with the company's internal systems. With this successful implementation, the logistics unicorn can focus on further enhancing its services and meeting the ever-increasing demands of its clients.
